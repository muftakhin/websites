Git

Базовые настройки

Команды

ls   //вывод списка файлов данной директории

ls -1   //вывод списка файлов, содержащихся в данной директори, в столбик

ls -1 name   //вывод в стольбик списка файлов директории name

mkdir name   //создание директории с именем name

cd name   //перейти в директорию с именем name

cd -   //перейти в предыдущую директорию

cd ~   //перейти в корневую директорию

cd ..   //перейти на уровень выше

cd ../../   //перейти на два уровня выше

pwd   //узнать естонахождение

start .   //открыть текущую папку

start ~/directory_1/directory_2/file.txt   //открытие файла file.txt находящемуся
                                                в директории directory_2

Установка и начальная настройка

После установки Git и создания аккаунта на удалённом репозитории нужно соединить локальный репозиторий с удалённым

Для первоначальной настройки Git зададим имя и адрес электронной почты (желательно такой же как и при регистрации на удалённом репозитории) 

user ~
$ git config --global user.name "First-name Second-name"

user ~
$ git config --global user.email "name@mail.com"

user - пользователь компьютера

Создадим директорию репозитория и перейдя в неё инициализируем git-репозиторий 

$ git init

Клонирование удалённого репозитория

user ~
$ git clone https://github.com/username/repositoryname.git

user ~ cd repositoryname
//Выдаст:

user ~/repositoryname master

add и commit

add - добавление файла в индекс изменений
commit - фиксирование изменений в локальном репозитории 

user ~/repositoryname master
$ git add Filename.txt

user ~/repositoryname master
$ git commit -m "Первый коммит"

Чтобы добавить все новые и изменённые файлы: 

$ git add .

push

push - отправка изменений на удалённый репозиторий 

user ~/repositoryname master
$ git push origin master

origin и master - значения по умолчанию
origin - короткое название ссылки на удалённый клонированный репозиторий
master - основная ветка репозитория

pull

pull - получение последнего состояния ветки и всех её предыдущих изменений с удалённого репозитория 

user ~/repositoryname master
$ git pull

Создание новой ветки

Создание новой ветки adaptive: 

user ~/repositoryname master
$ git checkout -b adaptive
//Выдаст:

user ~/repositoryname adaptive

git checkout -b - создание новой ветки
adaptive - название новой ветки

Переключение на другую ветку

Переключение на ветку second-branch: 

user ~/repositoryname master
$ git checkout second-branch
//Выдаст:

user ~/repositoryname second-branch

Внесение изменений в новую ветку

user ~/repositoryname adaptive
$ git add Filename.txt

user ~/repositoryname adaptive
$ git commit -m "Изменил что-то"

Отправка изменённой новой ветки

user ~/repositoryname adaptive
$ git push origin adaptive

Отправляем изменения из новой ветки adaptive локального репозитория в ветку adaptive удалённого репозитория

Работа с коммитами

Просмотр списка коммитов

user ~/repositoryname adaptive
$ git log --oneline

Просмотр схемы коммитов

user ~/repositoryname adaptive
$ git log --oneline --all --graph

Просмотр содержимого коммита

user ~/repositoryname adaptive
$ git cat-file -p индекс-коммита

Переключение на коммит

user ~/repositoryname adaptive
$ git checkout индекс-коммита

Создание указателя коммита

user ~/repositoryname adaptive
$ git checkout -b имя-указателя индекс-коммита

Работа с другого компьютера

Клонируем репозиторий на другой компьютер

other-user ~
$ git clone git@github.com:username/repositoryname.git

other-user ~
$ cd repositoryname
//Выдаст:

other-user ~/repositoryname master

Клонирование ветки master удалённого репозитория 

other-user ~/repositoryname master
$ git checkout origin/second-branch -b second-branch

Создание локальной ветки second-branch и получение в неё изменений из ветки second-branch удалённого репозитория

Откат последнего изменения

Просмотр текущего состояния

$ git status

• Покажет какие файлы изенены (modified:) или удалены (deleted:)
• Покажет так же на каком изменении находятся on branch master или on branch имя-указателя

Просмотр изменений

$ git diff

Возврат к состоянию крайнего коммита

$ git checkout folders-name/file-name.txt

Если прежде "закомиченый" файл file-name.txt был удалён, то он вернётся в своё прежнее состояние, т.е. восстановится

Слияние двух веток

user ~/repositoryname master
$ git merge second-branch

Сливание изменений из ветки second-branch в ветку master 

user ~/repositoryname master
$ git merge second-branch -m 'Коммит'

Слияние с созданием коммита

Cвязь локального репозитория с удалённым

Просмотр связей

user ~/repositoryname master
$ git remote -v

Добавление новой связи

user ~/repositoryname master
$ git remote add name-1 git@github.com:name-2/branch-2.git

В итоге, помимо связи c origin, мы увидим новую связь с name-1 репозиториями 

user ~/repositoryname master
$ git remote -v

//Выдаст

origin ... (fetch)
origin ... (push)
name-1 ... (fetch)
name-1 ... (push)

pull из добавленного репозитория

user ~/repositoryname master
$ git pull name-1 master

- забираем изменения к себе в master-ветку из master-ветки репозитория обозначенным как name-1
