Принципы каскадирования правил

Исхдный порядок

Приоритет получает правило, которое ниже в списке. Ибо позже интерпритируется. Это же относится и к наследуемым свойствам, тот, что ближе к элементу более приоритетен

Специфичность

Приоритет получает более специфичный селектор:
  tag = 1
  class = 10
  id = 100
  inline = 1000

Источник

• Авторское правило важнее пользовательского
• Пользовательсое правило важнее правила браузера

Первая строка или буква

        font:
        background:

        vertical-align:
        text-transform:
        line-height:

Селекторы атрибутов

[ ]
Селектор простых атрибутов (существования атрибута)

Соответствует конкретному атрибуту независимо от его значения 

элемент[атрибут]
img[title] {border: 3px solid;}
[ = ]
Селектор точного значения атрибута (равенства атрибута)

Соответствует конкретному атрибуту с конкретным значением 

элемент[атрибут="точное значение"]
img[title="first grade" {border: 3px solid;}
[ ~= ]
Селектор неполного значения атрибута (атрибута с пробелами)

Сответствует конкретному атрибуту, чьё значение присутствует в списке слов, разделённых пробелами (позволяет задать часть значения атрибута отделённую пробелами) 

элемент[атрибут~="часть значения отделённая пробелами"]
img[title~="grade"] {border: 3px solid;}
[ |= ]
Селектор разделённого дефисом значения атрибута

Позволяет задать часть значения атрибута, отделённую дефисами 

элемент[атрибут|="часть-значения-отделённая-дефисами"]
*[hreflang|="en"] {border: 3px solid;}
[ ^= ]
Селектор начала значения атрибута (префикса атрибута)

Соответствует конкретному атрибуту, чьё значение начинается с конкретной строки 

элемент[атрибут^="первая часть значения"]
img[src^="/images/icons"] {border: 3px solid;}
[ $= ]
Селектор конца значения атрибута (суффикса атрибута)

Соответствует конкретному атрибуту, чьё значение заканчивается конкретной строкой 

элемент[атрибут$="последняя часть значения"]
a[href$=".pdf"] {border: 3px solid;}
[ *= ]
Селектор части значения в любом месте атрибута (подстроки атрибута)

Соответствует конкретному атрибуту, чьё значение содержит конкретную подстроку 

элемент[атрибут*="любая часть значения"]

Форматирование текста

Свойство шрифта

Семейство шрифтов (Гарнитура шрифта)

• Свойство font-family даёт возможность задать список предпочтительных шрифтов
• Если имя шрифта состоит не из одного слова, то его указываем в двойных кавычках
• В конце списка всегда указываем название семейства шрифтов(serif, sans-serif, cursive или monospace)
• Вместо названия семейства шрифтов браузер будет применять тот шрифт, который определён им как используемый по умолчанию для указанного семейства шрифтов 

font-family: "Trebuchet MS", Verdana, sans-serif;

Размер шрифта

Конкретным размером
font-size: ...px;
         - ...em;
         - ...ex;
         - ...rem;

em - размер относительно наследуемого шрифта
rem - размер относительно шрифта в корневом элементе (<html>)

Процентным значением
font-size: ...%;
Используя абсолютное зарезервированное слово
font-size: xx-small;
         - x-small;
         - small;
         - medium;
         - large;
         - x-large;
         - xx-large;
Используя относительное (сравнительное) зарезервированное слово
font-size: larger;
         - smaller;

Дополнительный шрифты

Свойство @font-face позволяет использовать шрифт, даже если он не установлен на компьютере пользователя, предоставляя путь к файлу шрифта 

@font-face {
    font-family: "name";
    src: url{".../..."};
}

font-family: присваивает шрифту имя name 

@font-face {
    font-family: "ChunkFiveRegular";
    src: url{"fonts/chunkfive.eot"};
}
h1, h2 {
    font-family: ChunkFiveRegular, Georgia, serif;
}

Выравнивание строк текста

Интерлиньяж

Интерлиньяж - минимальное (ибо рисунок в строке его увеличит) расстояние между базовыми линиями строк текста.
Базовая линия - воображаемая линия, на которой располагаются нижние части символов. 

line-height: число (масштабный множитель на текущий размер;
           - ...em;
           - ...%;
           - normal;
           - inherit;

Можно воспользоваться свойством line-height, чтобы выровнять текст по средней линии блока: 

text-align: center;
line-height: (высота блока);

Красная строка

text-indent: ...px;
           - ...em; ()
           - ...% (на основе ширины родительского элемента);

Применяется к блочным элементам и ячейкам таблицы

Горизонтальное выравнивание

          - right;
          - center;
          - justify;
          - inherit;

justify определяет, что каждая строка абзаца, кроме последней, должна занимать всю ширину текстового поля.
Применяется к блочным элементам и ячейкам таблицы

Вертикальное выравнивание

vertical-align: baseline;
              - sub;
              - super;
              - top;
              - text-top;
              - middle;
              - bottom;
              - text-bottom;
              - px;
              - em;
              - %;

vertical-align выравнивает только inline-элементы и содержимое ячеек таблицы

Обработка пробелов между словами и переносы строк внутри элемента

Градиенты

Линейные градиенты

linear-gradient(color_1, color_2);

- простейший вариант 

linear-gradient(direction, color_1, color_2);

direction = to top
            to right
            to bottom
            to left
            to top right
            to top left
            to bottom right
            to bottom left
            ...deg
#banner {
    background-image: linear-gradient(180deg, yellow, green);
}

или

    background-image: linear-gradient(to bottom, yellow, green);
linear-gradient(direction, color_1 dot_1, ... ..., color_n dot_n);

dot = % пути по направлению (может быть со знаком " - ")
#banner {
    background-image: linear-gradient(90deg, yellow, orange 25%, blue);
}

- оранжевый появляется через 25% пути по линии градиента 

linear-gradient(yellow 50%, lightblue 50%);

Если точки перехода цвета совпадают, то один цвет резко сменит другой в этой точке перехода 

linear-gradient(yellow 50%, lightblue 0);

Если задать следующую точку равную 0, то она автоматически будет в том же месте, что и предыдущая (если меньше предыдущей, то на том же месте, что и предыдущая)

Блочная модель

Задавание размеров блока

• margin-"сторона": auto; занимает всё свободное пространство в этом направлении
• Если задать ширину блока, то margin, border и padding будут добавлены к его ширине и высоте 

width:
height: ...px;
      - ...%;
      - auto;
      - inherit;

• Применяется к block элементам и изображениям
• inline элементам width и height не задаются (не работают)
• % задаются от ближайшего родительского элемента с заданным аналогичным размером (если ни у одного из предков не задана ширина, отсчитывается от ширины окна браузера)

Задавание размеров области контента

Значения ширины и высоты применяются только к области контента.В результате размеры видимого блока будут складываться с размерами отступов, границ и полей

              - border-box;

}

    box-sizing: border-box;
}
.content-box-component {

Static

Обычное размещение в потоке

Relative

• Сдвигает блок относительно его начального положения в потоке
• Исходное пространство в потоке сохраняется и продолжает влиять на расположение окружающего контента
• Потенциально может перекрыть другие элементы

Absolute

• Элемент удаляется из потока и позиционируются относительно ближайшего родительского элемента не position: static;
• По умолчанию задано top: auto; right: auto; bottom: auto; left: auto;, если не задавать top, right, bottom или left, то элемент будучи вырванным из потока, окажется над тем же местом, которое он освободил
• Пространство, которое он занимал бы, смыкается
• Абсолютно позиционированные элементы не влияют на расположение окружающих элементов
• Если задать противоположные расстояния, то элемент растянется

• Элемент помещается относительно его ближайшего содержащего элемента
• html - начальный содержащий блок
• Абсолютно позиционированные объекты всегда ведут себя как блочные

Fixed

• Элемент удаляется из потока и позиционируется относительно окна браузера
• Смещение всегда относительно окна браузера

Sticky

top, right, bottom, left указывают границу, дальше которой элемент не будет перемещаться

Невозможно, когда высота позиционированного элемента равна высоте родительского элемента или значение его свойства overflow равна не visible

Все типы позиционирования (position:) ,кроме static делают inline элемент блочным (display: block;)

Порядок наложения

• z-index применяется элементам позиционированным (relative, absolute или fixed)
• По умолчанию элементы имеют значение свойства z-index: auto; и располагаются в том порядке,в котором появляются в документе (исходя из положения в HTML коде). Родительский элемент располагается ниже дочернего, а из дочерних более низко описаный в коде элемент располагается выше на странице 

z-index: число (целое);
       - auto;     (default)
       - inherit;

z-index - число, чем больше значение, тем выше располагается элемент (ближе к переднему плану),дочерние элементы наследуют z-index родителя. При равном значении z-index на передний план выходит тот элемент, который в коде HTML располагается ниже
• Родительский элемент не может располагаться выше дочернего элемента, какой бы z-index мы ему не присвоили. При этом значение z-index образует для вложенных элементов свой уровень иерархии внутри данного уровня. Так потомки одного элемента с указанным значением z-index никогда не будут выше потомков другого элемента с указанным более высоким значением z-index

• {flex-basis: } - базовый размер, к которому приводится размер flex-элемента.
