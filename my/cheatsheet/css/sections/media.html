Медиазапросы

В элементе link

<link rel="stylesheet" media="screen" href="name.css">

<link href="print.css" rel="stylesheet" media="print">

<link rel="stylesheet" media="screen and (orientation: portrait)" href="name.css">

<link rel="stylesheet" media="not screen and (orientation: landscape)" href="name.css">

Логику любого медиазапроса можно реверсировать, добавив not в его начало

В одной строке можно записать несколько выражений, где: and - логическое и, а , - логическое или 

<link rel="stylesheet" media="screen and (orientation: portrait) and (min-width: 320px)" href="name.css">

Можно создать список медиазапросов 

<link rel="stylesheet" media="screen and (orientation: portrait) and (max-device-width: 640px), projection" href="name.css">

Медиазапросы разделяются запятыми. Если после указания устройства отсутствуют в скобках значение:свойство, стиль применяется для всех устройств данного типа медиаустройств

С использованием @import

Добавляется в существующую таблицу стилей 

@import url("name.css") screen and (max-width:360px);

Код @import добавляется к HTTP-запросам, что влияет на скорость загрузки

Внутри CSS

@media device and (свойство: значение) {
    /* styles */
}

свойство - min
         - max-width
              -height
              -device-width
              -device-height
              -orientation
              -aspect-ratio
              -device-aspect-ratio
              -color
              -color-index
              -monochrome
              -resolution
         -scan
         -grid

device - all
       - print
       - screen
       - speach

"Контрольные точки" определяются не разрешением экранов устройств, а дизайном проекта
all and можно не указывать, если подразумеваются все устройства 

@media (min-width: 321px) and (max-width: 720px) {
    ...
}

Для устройств с высоким разрешением, использующих 2 точки на пиксель 

@media (min-resolution: 2dppx) {
    /* styles */
}
