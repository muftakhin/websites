Обработка ошибок

Не все ошибки можно "перехватить", например синтаксические или в асинхронно вызванных функциях

• Возникающая ошибка прерывает исполнение кода. Необходимо её обработать
• Код, который потенциально может выдать ошибку, необходимо заключить в оператор try catch 

try {
    action_1;
} catch (name_1){
    action_2;
}

action_3;

action_1 - код с потенциальной ошибкой. Выполняется, пока не возникнет ошибка
После возникновении ошибки выполнение кода прерывается и передаётся управление блоку catch и выполняется код action_2.
name_1 - объект содержащий информацию об ошибке, он называется исключением. Эту информацию можно вывести
action_2 - реакция на потенциальную ошибку.
После выполнения action_2 продолжается выполнение кода action_3. 

try {
    action_1;
} catch (name_1){
    console.table(name_1);
    action_2;
}
                
action_3;

Объект name_1 имеет 2 основных свойства: .name и message, значения которых тоже можно вывести 

try {
    action_1;
} catch (name_1){
    action_2;
    console.log(name_1.name);
    console.log(name_1.message);
}
                                
action_3;

Создание своих ошибок

action_1 ...
throw {name: 'string_1', message: 'string_2'};
... action_1;

или:

action_1 ...
throw new Error('string_3');
... action_1;

где:
string_1 - значение свойства name:
string_2 и string_3 - значения свойства message:

finally

try {
    action_1;
} catch (name_1) {
    action_2;
} finally {
    action_3;
}

• action_3 выполнится после action_1, если не выпадет ошибка в action_1.
• action_3 выполнится после action_2, если ошибка выпадет.
• Если внутри try имеется return, то он выполнится после finally.
